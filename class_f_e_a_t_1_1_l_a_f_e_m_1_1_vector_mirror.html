Detailed Description

template<typename DT_, typename IT_>
class FEAT::LAFEM::VectorMirror< DT_, IT_ >

Handles vector prolongation, restriction and serialization.

Template Parameters
DT_Data type in the vector
IT_Type for indexing

A Mirror handles the restriction of a given vector to a subvector of itself and the serialization into buffer vector (which always is a LAFEM::DenseVector), as well as the prolongation of such buffer vectors. The buffer vectors then can be communicated via MPI etc.

This can be used for restricting a vector associated with a Mesh to a subset of that Mesh, i.e. identified by a MeshPart. The standard use case is that we have an FE coefficient vector living on a patch that we want to communicate. For this, we need all coefficients (= vector entries) that contribute to other patches, and these are identified by a MeshPart called Halo. The mirror is constructed from the restriction (gather) and prolongation (scatter) matrices representing the adjacency structure of the underlying FE space and thus the coefficients.

Data survey:
_indices[0]: non zero indices
_scalar_index[0]: container size
_scalar_index[1]: non zero element count (used elements)
