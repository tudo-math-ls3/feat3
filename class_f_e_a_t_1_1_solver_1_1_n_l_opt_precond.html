Detailed Description

template<typename VectorType_, typename FilterType_>
class FEAT::Solver::NLOptPrecond< VectorType_, FilterType_ >

Abstract base class for preconditioners for nonlinear optimization.

Template Parameters
VectorType_The type of vectors the preconditioner acts upon.
FilterType_The type of filter that gets applied.

The main difference to SolverBase is that they have to offer a prepare() function that takes a state variable and prepares the preconditioner. An example would be that prepare() assembles the gradient matrix and apply() applies the inverse of the gradient matrix to the input by using a linear solver.
